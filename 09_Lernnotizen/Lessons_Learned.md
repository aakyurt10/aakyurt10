# Lessons Learned

## What worked well
- Small, focused lab scope made troubleshooting easier.
- Documenting every change reduced repeated mistakes.
- Testing on two clients helped validate GPO and DHCP behavior.

## Challenges
- DNS misconfiguration caused multiple follow-up errors.
- Group Policy timing led to false assumptions during tests.
- VM resource limits impacted performance in parallel tests.

## Improvements for next iteration
- Add a baseline checklist before each test cycle.
- Capture screenshots during each milestone.
- Introduce a second server role separation scenario.
