# Erkenntnisse

## Technisch
- AD, DNS und DHCP greifen stark ineinander. Ein DNS-Fehler wirkt sofort auf Domänenfunktionen.
- Eine klare OU-Struktur spart später viel Zeit bei GPO und Berechtigungen.
- Standardisierte Namenskonventionen reduzieren Verwechslungen im Betrieb.

## Methodisch
- Schrittweise Dokumentation direkt während der Umsetzung ist zuverlässiger als nachträgliches Schreiben.
- Vor jeder Änderung hilft ein kurzer Soll-Ist-Check.
- Wiederholbare Testschritte machen Troubleshooting deutlich schneller.

## Nächste Lernschritte
- Backup/Restore von Domain Controller (System State)
- Erweiterte GPO-Szenarien
- Monitoring-Grundlagen für Windows Server
